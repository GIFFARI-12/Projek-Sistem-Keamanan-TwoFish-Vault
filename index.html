1. Arsitektur Sistem Secara Keseluruhan

Sistem TwofishVault dirancang sebagai aplikasi web single-page (SPA) yang seluruh logika kriptografinya dieksekusi di sisi klien (browser). Tidak ada data yang dikirim ke server — plaintext, ciphertext, kunci, dan file diproses secara lokal menggunakan JavaScript murni. Antarmuka terdiri dari empat tab fungsional (Enkripsi Teks, Dekripsi Teks, Enkripsi File, Dekripsi File) yang berbagi input kunci dan mode operasi yang sama.

1Input Kunci (Hex 32/48/64)
2Validasi & Indikator Kekuatan
3Pilih Mode (ECB / CBC)
4Input Data (Teks / File)
5Proses Twofish Engine
6Output (Base64 / Blob Unduh)

Komponen utama sistem meliputi: TwofishEngine class (enkapsulasi seluruh logika cipher), fungsi padding PKCS#7, wrapper mode ECB/CBC, konverter hex & Base64, dan manajer file & DOM untuk interaksi antarmuka.

2. Key Schedule — Pembangkitan Subkey dan S-Box

Key schedule adalah proses paling kritis dalam Twofish. Dari kunci master (128, 192, atau 256 bit), sistem membangkitkan 40 subkey 32-bit dan 4 S-box yang bergantung pada kunci (key-dependent S-boxes). Proses ini dilakukan satu kali saat objek TwofishEngine diinstansiasi.

Langkah-langkah key schedule:

  1. Kunci master dipecah menjadi word 32-bit (little-endian) dan dipisahkan menjadi vektor Me (even words) dan Mo (odd words).
  2. Untuk setiap entri S-box (4 box × 256 entri), byte indeks dilewatkan melalui permutasi q0 dan q1 secara bergantian sebanyak jumlah word kunci, menghasilkan S-box yang unik untuk setiap kunci.
  3. 40 subkey dihitung menggunakan fungsi h yang melibatkan operasi penjumlahan dan XOR terhadap Me dan Mo, diikuti rotasi 8-bit pada subkey ganjil.
// Potongan kode: Key Schedule dalam TwofishEngine constructor(keyBytes) { this.keyBytes = keyBytes; this.keyLen = keyBytes.length; // 16, 24, atau 32 this.subkeys = new Array(40).fill(0); this.sBoxes = [[],[],[],[]]; this._keySchedule(); } _keySchedule() { const kwords = Math.floor(this.keyLen / 4); const Me = new Array(4).fill(0); const Mo = new Array(4).fill(0); // Pisahkan kunci menjadi even/odd words for (let i = 0; i < kwords; i++) { const word = (this.keyBytes[4*i] << 24) | (this.keyBytes[4*i+1] << 16) | (this.keyBytes[4*i+2] << 8) | this.keyBytes[4*i+3]; if (i % 2 === 0) Me[i/2] = word; else Mo[Math.floor(i/2)] = word; } // Bangkitkan S-box yang bergantung kunci for (let i = 0; i < 4; i++) { for (let j = 0; j < 256; j++) { let b = j; for (let k = 0; k < kwords; k++) { b = (q0[k % 4][b >>> 4] << 4) | q1[k % 4][b & 0xF]; } this.sBoxes[i][j] = b; } } // Bangkitkan 40 subkey for (let i = 0; i < 40; i += 2) { const A = this._f32(i, Me); let B = this._f32(i + 1, Mo); B = ((B << 8) | (B >>> 24)) >>> 0; // rotasi kiri 8 bit this.subkeys[i] = (A + B) >>> 0; this.subkeys[i + 1] = ((A + 2 * B) >>> 0); } }

3. Fungsi G dan MDS Matrix — Inti Non-Linear Twofish

Fungsi g adalah komponen non-linear utama dalam setiap round Twofish. Fungsi ini menerima satu word 32-bit sebagai input dan menghasilkan satu word 32-bit sebagai output melalui empat tahap:

  1. Word input dipecah menjadi 4 byte individual (b0 hingga b3).
  2. Masing-masing byte dilewatkan melalui S-box yang berbeda (S0 hingga S3), menghasilkan 4 byte output.
  3. Keempat byte tersebut dikalikan dengan MDS matrix 4×4 menggunakan operasi perkalian di Galois Field GF(28) dengan polinomial irreducible x8 + x6 + x5 + x3 + 1 (0x169).
  4. Hasil perkalian MDS digabungkan kembali menjadi satu word 32-bit.
// MDS Matrix (4x4) — nilai tetap dari spesifikasi Twofish const MDS = [ [0x01, 0xEF, 0x5B, 0x5B], [0x5B, 0xEF, 0xEF, 0x01], [0xEF, 0x5B, 0x01, 0xEF], [0xEF, 0x01, 0xEF, 0x5B] ]; // Perkalian di Galois Field GF(2^8) function gfMult(a, b) { let p = 0; for (let i = 0; i < 8; i++) { if (b & 1) p ^= a; a = (a & 0x80) ? ((a << 1) ^ 0x169) : (a << 1); b >>= 1; } return p & 0xFF; } // Fungsi g — transformasi non-linear inti Twofish _g(word) { const b0 = (word >>> 24) & 0xFF; const b1 = (word >>> 16) & 0xFF; const b2 = (word >>> 8) & 0xFF; const b3 = word & 0xFF; const s0 = this.sBoxes[0][b0]; const s1 = this.sBoxes[1][b1]; const s2 = this.sBoxes[2][b2]; const s3 = this.sBoxes[3][b3]; const outVec = mdsMultiply([s0, s1, s2, s3]); return (outVec[0] << 24) | (outVec[1] << 16) | (outVec[2] << 8) | outVec[3]; }

4. Struktur Round — Jaringan Feistel 16 Putaran

Twofish menggunakan jaringan Feistel 16 round dengan pre-whitening dan post-whitening. Setiap round memproses 4 word (x0, x1, x2, x3) masing-masing 32-bit, membentuk blok 128-bit. Operasi dalam satu round:

  1. Hitung t0 = g(x0) dan t1 = g(x1).
  2. Update x2 = x2 ⊕ (t0 + t1 + K2r+8) dan x3 = x3 ⊕ (t0 + 2t1 + K2r+9).
  3. Tukar posisi: (x0, x2) ↔ (x2, x0) dan (x1, x3) ↔ (x3, x1).

Pre-whitening melakukan XOR dengan subkey 0-3, post-whitening dengan subkey 4-7 (dengan urutan output yang di-swap).

encryptBlock(plainBlock) { let x0 = (plainBlock[0]<<24)|(plainBlock[1]<<16)| (plainBlock[2]<<8)|plainBlock[3]; let x1 = (plainBlock[4]<<24)|(plainBlock[5]<<16)| (plainBlock[6]<<8)|plainBlock[7]; let x2 = (plainBlock[8]<<24)|(plainBlock[9]<<16)| (plainBlock[10]<<8)|plainBlock[11]; let x3 = (plainBlock[12]<<24)|(plainBlock[13]<<16)| (plainBlock[14]<<8)|plainBlock[15]; // Pre-whitening x0 ^= this.subkeys[0]; x1 ^= this.subkeys[1]; x2 ^= this.subkeys[2]; x3 ^= this.subkeys[3]; // 16 round Feistel for (let r = 0; r < 16; r++) { const t0 = this._g(x0); const t1 = this._g(x1); x2 ^= (t0 + t1 + this.subkeys[8 + 2*r]) >>> 0; x3 ^= (t0 + 2*t1 + this.subkeys[8 + 2*r + 1]) >>> 0; [x0, x2] = [x2, x0]; // swap [x1, x3] = [x3, x1]; } // Post-whitening (output di-swap) x2 ^= this.subkeys[4]; x3 ^= this.subkeys[5]; x0 ^= this.subkeys[6]; x1 ^= this.subkeys[7]; // Output: x2 || x3 || x0 || x1 const out = new Uint8Array(16); new DataView(out.buffer).setUint32(0, x2, false); new DataView(out.buffer).setUint32(4, x3, false); new DataView(out.buffer).setUint32(8, x0, false); new DataView(out.buffer).setUint32(12, x1, false); return out; }

5. Mode Operasi — ECB dan CBC dengan IV Acak

Sistem mendukung dua mode operasi block cipher yang dapat dipilih pengguna melalui tombol radio:

  • ECB (Electronic Codebook): Setiap blok plaintext dienkripsi secara independen. Mode ini sederhana namun tidak menyembunyikan pola data. Cocok untuk pengujian dan pembelajaran.
  • CBC (Cipher Block Chaining): Setiap blok plaintext di-XOR dengan ciphertext blok sebelumnya sebelum dienkripsi. Blok pertama menggunakan Initialization Vector (IV) acak 16 byte yang di-generate oleh crypto.getRandomValues(). IV disimpan di depan ciphertext (prepend) dan digunakan kembali saat dekripsi.
// Proses enkripsi dengan dukungan ECB & CBC function processEncrypt(bytes, keyBytes, mode) { const engine = new TwofishEngine(keyBytes); const padded = pkcs7Pad(bytes, 16); let prev, iv; if (mode === 'CBC') { iv = crypto.getRandomValues(new Uint8Array(16)); prev = iv.slice(); } else { iv = new Uint8Array(16); prev = new Uint8Array(16); } const blocks = []; for (let i = 0; i < padded.length; i += 16) { let block = padded.slice(i, i + 16); if (mode === 'CBC') { for (let j = 0; j < 16; j++) block[j] ^= prev[j]; } const enc = engine.encryptBlock(block); blocks.push(enc); if (mode === 'CBC') prev = enc.slice(); } const body = concatU8(...blocks); return mode === 'CBC' ? concatU8(iv, body) : body; }
